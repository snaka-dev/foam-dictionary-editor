# リリースノート

## v1.5.0 — 2026-06-10

### 改善

**BlockMesh ビューア：サイドバイサイドモード**

- `blockMeshDict` がアクティブなファイルで BlockMesh パネルが利用可能なとき、上部タブウィジェットの右上に **⊞** トグルボタンが表示されます。
- クリックするとツリーの右隣に 3D ビューが水平スプリッターで並列表示され、ツリーの編集と 3D ビューを同時に確認できます。サイドバイサイドモード中は独立した **BlockMesh** タブが非表示になり、モードを切ると復元されます。
- 3D ビューは自動更新されません。ツリーを編集した後は BlockMesh パネルの **Refresh** をクリックしてください。
- xterm ターミナルがアクティブな間はボタンが無効化されます（GPU/OpenGL 競合のため）。

**比較パネル：比較モード以外では非表示に**

- 参照ケースが読み込まれていないときは、参照ケースのツリーペインが完全に非表示になりました。従来はスプリッターの非表示スロットとして存在し、メインツリーと詳細ペインの間に細いスプリッターハンドルの隙間が残っていました。
- ペインは差分バーで **Side by side** にチェックを入れたときに表示され、チェックを外すか **Clear** をクリックすると非表示になります。

**BlockMesh ビューア：変数ベースメッシュのプレビューモード**

- `vertices` ブロックに変数参照（`$varName`）が含まれる場合、頂点テーブルの X/Y/Z セルはデフォルトで読み取り専用になりました（直接書き込みは無言で失敗しており、誤解を招いていました）。
- Vertices パネルの上部（グループボックス内、メインツールバーではない）に **⚙ Variable-based** チップと **Preview** トグルボタンが表示されます。**Preview** をクリックするとプレビューモードになり、テーブルセルが編集可能になって各変更が即座に 3D ビューへ反映されますが、ツリーとファイルは変更されません。プレビューモード中は黄色いバナーが表示されます。
- **Refresh** をクリックするとプレビューモードを終了してツリーの頂点座標に戻ります。
- 変数参照を含まないメッシュでは従来どおり、テーブルへの編集がツリーとエディタに即座に反映されます。

**BlockMesh ビューア：多段変数チェーンの解決**

- `blockMeshDict` 内の任意の深さの変数チェーンを完全に解決できるようになりました。従来は `z001 $z1;` のようなマクロ参照が `z1 #eval{$z0+$dz0};` で定義されていた場合、マクロパスが `#eval` 評価より先に実行されるため、数値ではなく生の式文字列が残っていました。
- 固定の 3 パス方式を、マクロ参照の解決と `#eval` 式の評価を交互に繰り返す反復ループに置き換えました。反復上限はトップレベルの変数定義の数であり、これは非循環依存グラフの理論的な最大深度に等しいです。
- 循環参照（`a $b; b $a;` など）は、無限ループやクラッシュを起こすことなく未解決のまま静かにスキップされます。

## v1.4.0 — 2026-06-09

### 新機能

**foamMonitor ランチャー**

- トップバーの **foamMonitor…** ボタンからダイアログを開き、`foamMonitor` を起動して残差などのデータを gnuplot でプロットできます。
- ダイアログではファイル（ソルバーログや `postProcessing/` の出力ファイル）の選択、オプション（対数スケール、グリッド、リフレッシュ間隔、アイドルタイムアウト）の設定、追加オプションの自由入力ができます。
- foamMonitor の実行中はボタンが **■ foamMonitor** に変わり、クリックするとプロセスを停止します（foamMonitor シェルと gnuplot ウィンドウの両方を終了）。新しいケースを開いたときも実行中のインスタンスは停止されます。
- 新しい gnuplot バージョンで非推奨となった `reread` コマンドを起動時に自動的にパッチし、モダンな `load ARG0` 相当に置き換えます。gnuplot のバージョンに関係なくウィンドウが正しくリフレッシュされます。
- 指定したファイルが存在しない場合や foamMonitor がエラーで終了した場合は警告ダイアログを表示します。

**BlockMesh ビューア：変数と `#eval` 式のサポート**

- `blockMeshDict` 冒頭の変数定義（例：`xMax 0.5;`、`nCell 20;`）が、3D ジオメトリ抽出前に `vertices` および `blocks` セクションで置換されるようになりました。`$varName` および `${varName}` の両参照形式に対応しています。
- 他の変数を参照するマクロ変数（`nx $nCell;`）を 1 レベル深まで解決します。
- 算術式（`zMax #eval{ $length / $nCell };`）は変数置換後に評価されます。対応演算子：`+`、`−`、`*`、`/`、括弧。
- 従来は未解決の `$variable` 参照により `float()` 変換が失敗し、頂点が欠落したり、最悪の場合は範囲外の頂点インデックスによる VTK クラッシュが発生していました。
- **レキサー修正：** 内部にスペースのない `#eval{expr};` 形式が、セミコロンを含む単一のディレクティブトークンとして解析され、パーサーが直前の変数の値として `vertices` ブロック全体を消費してしまう問題を修正しました。レキサーは `{` でディレクティブトークンの読み取りを停止するようになり、深さ追跡が正しく機能します。

**ドラッグ＆ドロップによるケースオープン**

- ファイルマネージャからケースディレクトリをウィンドウ上の任意の場所にドロップするとケースを開きます。ツリービュー、エディタ、ファイル一覧など、すべてのパネルがドロップ対象として有効です。
- 現在開いているファイルに未保存の変更がある場合は、新しいケースを読み込む前に確認ダイアログが表示されます。

## v1.3.0 — 2026-06-01

### 改善

**ケース比較の高速化・ノンブロッキング化**

- ファイル一覧の差分件数をイベントループ 1 ティックごとに 1 ファイルずつ段階的に計算するようになりました。UI はブロックされず、すべてのファイルの解析完了を待たずに `≠` マーカーが順次表示されます。
- `FoamFile` ヘッダーを持たない大きなファイル（`log.simpleFoam` や残差出力など）は、差分計算および境界条件パネルの読み込み時に自動的にスキップされます。100 KB 未満のファイルは常に解析を試みるため、`FoamFile` ブロックを持たない小さなカスタムソルバー辞書は引き続き正常に動作します。
- 大きな非ディクショナリファイルを開く際に確認ダイアログが表示されるようになりました。ツリービューが利用できないこと、および読み込み中にアプリケーションが応答しなくなる可能性があることを警告します。確認後はステータスバーにメッセージが表示されます。

### 新機能

**UI 言語選択（英語 / 日本語）**

- **Settings > Language** サブメニューを追加。**English** または **日本語** を選択するとアプリケーションが選択言語で再起動します。
- 言語設定は `app_config.json` に保存され、起動時に適用されます。デフォルトは英語です。
- 新規 `i18n/` モジュール: `tr()` 関数がアクティブな言語辞書を参照し、未翻訳のキーは英語文字列へ自動フォールバックします。
- 新しい言語（例: イタリア語）の追加は `i18n/<コード>.py` 翻訳ファイルを1つ作成するだけで完了し、他のコード変更は不要です。

## v1.2.0 — 2026-05-30

### 新機能

**Boundary ビュー — テーブルのコピー**

- Boundary パネルのツールバーに **Copy Table** ボタンを追加。ドロップダウンメニューで 2 つの形式を選択できます:
  - **Copy as Markdown** — GitHub Flavored Markdown のパイプテーブル。Lines per cell が 2 以上の場合、複数行セルは `<br>` タグで結合され GitHub 上で改行として表示されます。
  - **Copy as CSV** — RFC 4180 準拠の CSV。複数行セルは引用符付きフィールドとして保持され、Excel・LibreOffice Calc で正しく表示されます。
- どちらの形式も行ヘッダー列を含み、転置状態を反映します。

**ケース比較 — サイドバイサイド表示**

- 差分バーの **Side by side** トグルで中央パネルを左右に分割します。左ペインは編集可能な現在のケースのツリーを表示し、右に新しい **Reference** ペインが開いて対応するファイルを読み取り専用で表示します。
- 参照ケースのツリーでは、参照にしか存在しないキーを**薄緑色**でハイライト表示します。
- 参照ペインの任意のリーフノードを右クリックして **Use this value** を選択すると、対応するノードの値を現在のケースに即座に適用できます。適用後、差分ハイライトは自動的に更新されます。
- ファイル一覧の **Changed files only** チェックボックス: 差分のないファイルを非表示にし、参照ケースと異なるファイルのみを表示します。
- 差分マーカー（`≠N` / `≠0`）は、比較開始時に全ファイルに対して即座に計算されるようになりました（従来はファイルを開くたびに遅延計算していました）。

### 改善

**BlockMesh パネル**

- ツールバーを 1 行にコンパクト化しました。
- STL に加えて OBJ ジオメトリファイルの読み込みとオーバーレイ表示に対応しました（`Load STL / OBJ…` / `Clear STL`）。

**ファイル一覧**

- 追加ファイルのボタンを常に表示するようにしました（追加ファイルが登録されている場合は件数を表示）。

**ケース比較**

- 差分バーに **Side by side** トグルとカラーレジェンドを追加しました。

## v1.1.0 — 2026-05-23

本リリースでは、インタラクティブな BlockMesh 3D ビューア、差分オーバーレイによるケース比較、ケース全体にわたる境界パッチのリネーム、Boundary パネルからエディタへのナビゲーション、ツリー ↔ エディタのソース行同期など、複数の主要機能を追加しました。また、Windows 向けの起動バリアント（no-terminal）、追加ディレクトリの再帰スキャンオプション、snappyHexMeshDict スキーマの拡張、各種エディタおよびユーザビリティの改善も含まれています。

### 新機能

**BlockMesh 3D ビューア** *（`pyvista` および `pyvistaqt` が必要）*

- 新しい **BlockMesh** タブで `blockMeshDict` のジオメトリをインタラクティブに 3D プレビュー。頂点（赤い球）、ヘックスブロックのエッジ（ワイヤーフレーム）、境界パッチ面（パッチ種別で色分け: wall、inlet、outlet、symmetry など）を表示。
- 頂点、頂点ラベル、ブロックエッジ、ブロックラベル、境界面、軸、グリッド、寸法テキストの表示切替チェックボックス。
- **Block labels** — 各ヘックスブロックの重心にブロックインデックスを表示（濃青テキスト）。頂点ラベルと同様の機能。
- **Color blocks** — 各ヘックスブロックを定性的パレット（tab10）の異なる色で表示。ワイヤーフレームとソリッド面の両方に適用。
- **Solid blocks** — 半透明のソリッドヘックスブロック面（opacity 0.25）をワイヤーフレームに加えて表示。Color blocks のパレットを共有。
- **Label size** — 第 2 ツールバー行の共用スピンボックス（範囲 6–32、デフォルト 10）で頂点ラベルとブロックラベルのフォントサイズを一括調整。
- 視点方向ボタン（+X / −X / +Y / −Y / +Z / −Z / Iso）でカメラを素早く切り替え。
- パネル下部のマウス操作ヒントバー（ドラッグ = 回転、Shift+ドラッグ = 平行移動、スクロール = ズーム）。ホバーで詳細ツールチップ表示。**Help > Keyboard Shortcuts…** でも確認可能。
- 3D ビューと並んで表示される頂点テーブル（インデックス | X | Y | Z）。行クリックで頂点をハイライト、座標セルをダブルクリックで値を編集 — 変更は FoamNode ツリーとテキストエディタに即座に反映。
- STL ジオメトリファイルの読み込みとオーバーレイ表示（`Load STL…` / `Clear STL`）。
- **View > BlockMesh 3-D Panel**（チェック式）でターミナルモードとは独立してタブを表示・非表示に切り替え可能。xterm が有効なときはアクションが無効化され、GPU コンフリクトの理由を示すラベルに変わります。
- `no-terminal-blockmesh` バリアントでは常時表示。`standard` バリアントではターミナルがシンプルモードのときのみ表示。

**ケース比較**

- **Case > Compare with Case…** — 参照ケースのディレクトリを選択し、現在のケースと比較。
- アクションバー下の差分バーに参照パスとカラーレジェンドを表示。**Clear** をクリックすると比較モードを終了。
- ツリーオーバーレイ: 薄黄色 = 値の変更、薄青 = このケースのみに存在するキー。ハイライト行をホバーするとツールチップに参照値を表示。
- ファイル一覧のマーカー: `≠N`（アンバー）= 差分あり、`≠0`（グレー）= 確認済みで一致、マーカーなし = 未訪問。50 件超は `≠50+` に丸め。ファイルを開いたタイミングで遅延更新。

**境界パッチのリネーム（Rename Boundary）**

- **Boundary** パネルのセル・パッチ列/行ヘッダー、またはツリービューのパッチノードを右クリックして **Rename Boundary…** を選択。
- ダイアログにパッチ名が含まれるすべての読み込み済みファイル（`blockMeshDict` の境界エントリおよびフィールドファイルの `boundaryField` パッチキー）が一覧表示されます。更新するファイルを選択し、新しい名前を入力して **Rename** をクリック。選択したすべてのファイルが一括でメモリ上に反映され、ダーティマークが付きます。

**Boundary パネルのシングルクリックナビゲーション**

- Boundary パネルのセルをクリックすると、対応するフィールドファイルをエディタで開き、該当パッチエントリにスクロールしてアンバーハイライトで表示。**Auto-scroll editor** チェックボックスでこの動作を切り替えられます。
- クリックしたセルが現在開いているファイルと異なる場合、ファイル一覧とエディタが自動的に更新されます。

**ツリー ↔ エディタ ソース行同期**

- ツリーノードをクリックすると、対応するソース行がテキストエディタでアンバー背景でハイライトされ、スクロールします。
- **Find in Tree** ボタン（または `Ctrl+Shift+T`）で現在のエディタカーソル行のソース範囲をカバーする最も深いツリーノードを選択します。
- エディタテキストが未保存の間（行番号が古い状態）はジャンプが無効化され、チェックボックスのラベルとツールチップで状態を表示します。

**ツリーキーフィルター**

- ツリー上部のフィルターバーで大文字小文字を区別しないキーワード絞り込みが可能。子孫ノードが一致する場合は親ノードも表示されます。

**ケースの再読み込み（Case Reload）**

- **Case > Reload Case** — メモリ上の編集をすべて破棄し、ディスクからケースを再読み込み。未保存ファイルがある場合は件数を示す確認ダイアログを表示してから実行します。

**`0/` ディレクトリの削除**

- ファイル一覧の `0` グループヘッダーを右クリックして **Delete '0' directory…** を選択すると、`0/` ディレクトリをディスクから完全削除。`0.orig` が存在するときのみ表示されます。削除前にフルパスを示す確認ダイアログを表示します。

**追加ディレクトリの再帰スキャン**

- 追加ディレクトリをフラットスキャン（直下ファイルのみ、デフォルト）または再帰スキャン（`rglob` でサブディレクトリ含む）に設定できるようになりました。
- **Settings > Manage Extra Files & Directories…** > **Extra Directories** タブでエントリを選択し **Toggle Recursive** をクリックすると再帰スキャンを有効・無効に切り替えられます。有効なエントリは `[recursive]` と表示されます。
- サブディレクトリにファイルを含む結果ツリー（例: `validation/`）などに便利です。

**アプリケーションバリアント**

- `--variant` フラグで起動設定を選択: `standard`（ターミナル + BlockMesh）、`no-terminal`（Windows 向け）、`no-terminal-blockmesh`（BlockMesh 常時表示・ターミナルなし）。選択したバリアントは `app_config.json` に保存され、次回以降の起動時に自動適用されます。
- ターミナルモードの切替（xterm ↔ シンプル）は Terminal タブ内のチェックボックスで行います。シンプルモードに切り替えると BlockMesh タブが表示されます。

**ツリーのコピー＆ペースト**

- ツリーパネルで `Ctrl+C` / `Ctrl+V` によるツリーエントリのコピー＆ペーストが可能。ショートカットはツリーウィジェットにスコープされており、下部のテキストエディタとは干渉しません。

### 改善

**View メニュー**

- **View > Show Type Column** — ツリービューの Type 列を表示・非表示に切り替え。
- **View > BlockMesh 3-D Panel** — パネルの表示切替（上記参照）。

**エディタツールバー**

- **Find Prev**（Shift+F3）と **Find Next**（F3）ボタンを追加。
- **Find in Tree** ボタン（`Ctrl+Shift+T`）— 現在のエディタ行から対応するツリーノードにジャンプ。
- ツールバー右側に **Line: N** として現在のカーソル行番号を表示。

**キーボードショートカットダイアログ**

- **Help > Keyboard Shortcuts…** ですべてのキーボードショートカットを一覧表示。
- 新しい **BlockMesh 3-D viewer (mouse)** セクションで回転・平行移動・ズーム・カメラリセット・フライトゥーポイントの操作方法を確認できます。

**snappyHexMeshDict スキーマ**

- `snappyHexMeshDict` のスキーマカバレッジを拡張: `snapControls`、`addLayersControls`、`meshQualityControls` のキーを追加。
- ユーザー定義名のサブ辞書（ジオメトリエントリ、細分化サーフェス/リージョン、レイヤーパッチグループ）に対して祖父キーコンテキストを使ったスキーマ検索に対応。ユーザー定義名に関わらず正しいスキーマが表示されます。

**パーサー**

- 括弧ブロック内（頂点リストなど）のインライン `//` コメントが無音で破棄されていたバグを修正。コメントはブロック内の `unknown_raw_entry` ノードとして保持されるようになりました。

**Boundary パネル**

- セルおよび列/行ヘッダーの右クリックメニューに **Rename Boundary…** を追加。
- パッチ列/行ヘッダーを右クリックして全フィールドファイルへの一括パッチ追加・削除が可能。

**ファイル管理**

- パーサーが完全に解釈できなかったエントリ（`unknown_raw_entry` ノード）をツリーでアンバー色でハイライト表示し、通常エントリと区別しやすくなりました。

## v1.0.0 — 初回公開リリース

初回公開リリース。主要機能:

- OpenFOAM 辞書ファイルの構造化ツリービューおよびテキストエディタ。
- `TARGET_FILES`、`0/`、`0.orig/` を自動スキャンするファイル一覧。追加ディレクトリの登録も可能。
- 境界条件テーブル（閲覧・編集・作成・削除・コピー・ペースト・全ファイル一括追加/削除）。
- `controlDict`、`fvSchemes`、`fvSolution`、`blockMeshDict`、`snappyHexMeshDict` の組み込みスキーマヘルプ。
- フル PTY xterm.js ターミナル（Linux/macOS）。ケースを開くと自動でそのディレクトリへ移動。
